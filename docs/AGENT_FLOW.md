# Agent 架构流程图

## 总体架构

```
┌──────────┐     ┌──────────────┐     ┌─────────────────┐
│ LLM Agent│────▶│  SCA 守卫     │────▶│  诊断管道         │
│ (决策层)  │     │  (约束层)     │     │  evaluate()       │
│          │     │              │     │  module_ablation() │
│          │     │              │     │  analyze_task()   │
└──────────┘     └──────────────┘     └─────────────────┘
      ▲                  │                      │
      │                  ▼                      │
      │          追问/重复检测/停滞              │
      │                                        │
      └──────────── 工具调用结果 ───────────────┘
```

**LLM Agent** 决定调用哪个工具、用哪些参数。
**SCA 守卫** 在 Agent 决策和管道执行之间——不是拦截操作，是注入追问。
**诊断管道** 执行实际的 CFD 特征选择 + 分类器评估。

## Agent 探索流程

```
┌──────────────────────────────────────────────────┐
│ Step 1 · 观察 (analyze_task)                      │
│   Agent 获取 120 维权重分布、模块报告              │
│   → 形成"哪个模块可能重要"的初始判断                │
└──────────────────────────────────────────────────┘
                     ↓
┌──────────────────────────────────────────────────┐
│ Step 2 · 消融 (module_ablation)                   │
│   Agent 逐个/批量关闭模块, 观察精度变化             │
│   → 发现关键事实: "关 time 后精度大涨"              │
│   → 形成假设: "time 模块是跨域噪声"                │
└──────────────────────────────────────────────────┘
                     ↓
┌──────────────────────────────────────────────────┐
│ Step 3 · 验证 (test_subset)                       │
│   Agent 用 base_dims (去 time 的 108 维) 选维→分类 │
│   → 验证假设: 精度确实超全维基线                   │
│   → 探索不同 selector/n_dims/分类器组合            │
└──────────────────────────────────────────────────┘
                     ↓
┌──────────────────────────────────────────────────┐
│ Step 4 · 收敛                                     │
│   Agent 在探索中锁定最优组合                       │
│   SCA 追问"天花板?"时 → Agent 诊断是否触顶         │
└──────────────────────────────────────────────────┘
```

## SCA 守卫的工作方式

SCA 不拦截工具调用——它**在 Agent 做出选择时注入追问**：

| 触发条件 | SCA 追问 | 目的 |
|---|---|---|
| 消融只做了 1-2 个模块就选维 | `缺消融: band,cepstral. 先消融?` | 防止在不了解模块结构的情况下盲搜 |
| 重复调用相同 (selector, n_dims) | `weighted+20d 已试 (0.572). 区别?` | 防止浪费时间重复无效组合 |
| 连续 3 轮未突破最优 | `3 轮未破 0.833. 极限?` | 促使 Agent 诊断是否已触及特征池天花板 |
| 只用了 1-2 种 selector | `未试:fisher,l1_lasso. 凭什么更差?` | 打破路径依赖, 尝试不同原理的算法 |

**SCA 不告诉 Agent "该做什么"——只问"为什么这样做？别的选择考虑过吗？"**

---

## 关键设计决策

**SCA 是"软约束", 不是"硬拦截"。**

对比 [embodied-ai-robot](https://github.com/kanglongh/embodied-ai-robot) 的硬件 SCA：
- 硬件 SCA（机器人）：舵机角度越界 → 硬钳位到安全范围。物理后果不可逆，必须硬拦。
- 软件 SCA（诊断 Agent）：选维重复/消融不足 → 注入追问。探索行为可逆，软追问更合适。

两者共享同一个设计基因——不信任 LLM 的自律，用外部约束兜底。

---

## 实验验证

`logs/` 目录下有两份对照实验日志（SCA ON vs SCA OFF），证明了：

1. Agent 在没有 SCA 的情况下也能独立完成假设→验证的科学循环
2. SCA 成功推 Agent 尝试了原本不会尝试的算法（l1_lasso）
3. SCA 的追问存在过度触发问题——碎片化了探索主线
4. 最终精度天花板由 CFD 特征表达力决定，不由搜索策略决定

详见 [logs/README.md](../logs/README.md)。
